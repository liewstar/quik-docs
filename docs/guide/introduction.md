# 简介

## 什么是 Quik？

**Quik** 是一个基于 Qt 的响应式 XML UI 框架，让你可以用简洁的 XML 语法声明式地构建 Qt 界面，无需学习 QML。

### 核心特性

- **XML 声明式 UI** - 用 XML 定义界面结构，直观易懂
- **双向数据绑定** - 变量和 UI 自动同步
- **条件表达式** - 支持 `visible`/`enabled` 条件控制
- **类型安全** - `QuikViewModel` 提供类型安全的变量访问
- **轻量级** - 纯 C++ 实现，编译为静态库

## 为什么选择 Quik？

### 对比传统 Qt Widgets

| 特性 | 传统方式 | Quik |
|------|---------|------|
| UI 定义 | C++ 代码或 Qt Designer | XML 声明式 |
| 数据绑定 | 手动连接信号槽 | 自动双向绑定 |
| 条件显示 | 手动管理 | 表达式自动控制 |
| 代码量 | 较多 | 精简 |

### 对比 QML

| 特性 | QML | Quik |
|------|-----|------|
| 学习成本 | 需要学习新语言 | XML + C++ |
| 运行时 | 需要 QML 引擎 | 纯 C++ |
| 类型安全 | JavaScript 动态类型 | C++ 静态类型 |
| 集成难度 | 需要额外配置 | 直接链接静态库 |

## 适用场景

Quik 特别适合以下场景：

- **工具软件** - 参数面板、设置对话框
- **CAD/CAE 软件** - 属性编辑器、功能面板
- **快速原型** - 快速构建交互界面
- **已有 Qt 项目** - 无缝集成到现有代码
